## REST API 의 구성


- 자원(resource) -> URI
- 행위(verb) -> HTTP Method
- 표현(Representations)


## REST 의 특징

- Uniform
  - uniform interface 는 URI 로 지정된 리소스를 통일, 한정적 인터페이스로 수행하는 아키택처 스타일
- Stateless
  - 무상태성, 세션 정보나 쿠키정보를 별도로 저장 관리하지 않음 -> 구현의 단순화
- Cacheable
  - 
- Self-descriptiveness
- Client-Server
- 계층형 구조


## REST API 설계의 핵심
1. URI는 정보의 자원을 표현해야한다
2. 자원에 대한 행위는 HTTP Method(GET, POST, PUT, DELETE)로 표현한다
